Projete uma simulação de um ambiente de uma casa. A casa deverá ter uma quantidade de cômodos
(limitados a 10) os quais devem ser definidos pelo usuário. Cada cômodo tem uma cor, interruptores para
acender lâmpadas, pelo menos uma porta, ter ou não um ar-condicionado (caso possua, deverá poder ser
acionado via interruptor). As portas só podem ser formadas a partir de um tipo de fechadura, as quais podem:
simples ser abertas com chave apenas; ou inteligente ser abertas com chave ou senha. Cada fechadura só
possui uma chave correta, mas duas fechaduras inteligentes podem possuir a mesma senha de abertura. Cada
objeto da casa deverá ter um método que apresente as informações do objeto. A casa deverá ser visitada por
pessoas (identificadas através de um nome), as quais são as responsáveis por acionar os componentes da
cada. Crie o cenário de teste para validar a implementação das suas classes. As ações executadas para testar
o funcionamento das classes devem ser verificadas no terminal. Por exemplo: um método acender lâmpada
deve informar que a lâmpada foi acessa; um método abrir porta deve informar que uma porta foi aberta e a
qual cômodo pertence, e assim por diante. Elabora o Diagrama de Classes UML para demonstrar o
relacionamento entre as classes conforme a descrição desse contexto e para servir de base para a
implementação do programa.
